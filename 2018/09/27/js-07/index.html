    shim&polyfill

  • 提供API来在旧的基础之上支持类似的新标准的功能
  • polyfill 更贴近标准，扩展原生对象来提供几乎一样的新标准支持

    语法

    let 与 const

  1. let和const不可以重复声明

  2. let和const都在块级作用域有效

    箭头函数

  3. 箭头函数只能替换函数表达式，不能替换函数声明，因为它是匿名的

  4. 不能赋值常量作为构造函数
  5. this指向是在定义时决定的而不是之前普通函数调用时定义的，this取决于父执行上下文的this
